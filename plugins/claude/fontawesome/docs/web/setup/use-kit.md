# Use a Kit

Source: https://docs.fontawesome.com/web/setup/use-kit

## What's a Kit?

A Font Awesome Kit is "the place to manage all your icons for a project." Kits offer:

- Auto or custom subsetting
- Support for free or Pro/Pro+ icons
- Ability to upload custom icons
- Choices between web font or SVG rendering
- JavaScript or CSS-only embedding options

## Creating and Using a Kit

Access kits through account page or Kits Listing page. New kits require selecting:

1. Icon styles (Free or Pro/Pro+)
2. A delivery method (Hosted, Self-Host, Package, or Desktop)
3. Kit name and appearance customization

## Delivery Methods

### Hosted

Single line of code in your project's `<head>`. Simplest option.

### Self-Host

Pro users only; download and manage files yourself.

### Package

npm or yarn for node-based projects.

### Desktop Download

SVGs and OTF files for desktop applications.

## CSS-Only Embedding

For projects avoiding JavaScript, kits support "CSS Kit embed code" requiring v7 with Web Fonts. This approach "blocks the page from loading until the icons are ready."

## Kit Settings and Customization

- **Subsetting choices** — auto or custom
- **Style selection** — pick only the styles you need
- **Version specification** — pin or use latest
- **Domain limiting** — restrict where the kit can load
- **Conflict detection** — detect other FA versions on page

Custom subsetted kits include "ONLY the icons you've selected plus any custom icons."

## Downloading and Performance

Pro kits enable downloading for self-hosting. Downloads include:

- CSS
- JavaScript
- SCSS
- SVG sprites
- Webfonts
- Metadata folders

Auto-subsetting "takes just the icons you're using in your project and creates a subset on the fly."

## Pageviews and Limitations

Monthly pageviews counted across all subscriptions. Exceeding limits may incur overage fees or require plan upgrades. Domain limiting helps control unauthorized usage.
