# shadcn/ui Introduction

## Overview

shadcn/ui is a set of beautifully-designed, accessible components and a code distribution platform that works with multiple frameworks and AI models.

**Key Philosophy:** This is NOT a traditional component library. It provides the foundational approach for constructing your own component system.

## Core Principles

### Open Code
The component code is directly accessible to developers. You receive the actual implementation rather than a compiled package, enabling complete customization without wrapper components or style overrides.

### Composition
Every component shares a consistent, composable interface pattern. When new components are introduced, they follow the same API design and styling conventions, creating predictability across the entire system.

### Distribution
shadcn/ui functions as a code distribution platform using:
- **Schema:** A flat-file structure defining components, dependencies, and properties
- **CLI:** Command-line tooling for cross-framework component distribution

### Beautiful Defaults
Components arrive with thoughtfully selected default styling that works cohesively.

### AI-Ready
The open architecture and standardized APIs allow language models to understand, analyze, and generate components that integrate seamlessly with your existing design system.

## Traditional vs. Open Code Approach

Conventional libraries require installing packages and using pre-built components. This workflow becomes problematic when customization needs arise or when specific components are unavailable.

The shadcn/ui model eliminates these friction points by giving developers direct access to component source code for modification and extension.
