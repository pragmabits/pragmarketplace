# Icon Design Guidelines

Source: https://docs.fontawesome.com/web/add-icons/upload-icons/icon-design

## The Canvas

The foundational element is the Icon Canvas, which measures **20 x 20 pixels** as a standard square grid system. This ensures uploaded icons render consistently alongside official Font Awesome icons.

Templates are available for:
- **Adobe Illustrator**: https://fontawesome.com/files/font-awesome-icon-template.ait
- **Figma**: https://www.figma.com/community/file/1068599043365999834

## Visual Balance — Keyshapes

Four core silhouettes guide icon design:

| Keyshape | Dimensions |
|----------|-----------|
| Square | 14 x 14 pixels |
| Circle | 16 pixel diameter |
| Rectangle (Vertical) | 12 x 16 pixels |
| Rectangle (Horizontal) | 16 x 12 pixels |

## Stroke Weight Standards

| Style | Stroke Weight |
|-------|--------------|
| Solid | Filled shapes with 2px (structure) and 1.5px (detail) |
| Regular | 1.5 pixels |
| Light | 1 pixel |
| Thin | 0.5 pixel |

## Corner and Endcap Treatment

- **Classic family**: Rounded corners
- **Sharp family**: Mitered (sharp) corners
- Consistent endcap styling prevents visual inconsistency across icons

## Overlaps and Cutouts

Cutouts around shapes extend out 1.5 pixels for added legibility across all Font Awesome styles. The standard slash element maintains a 1.5-pixel gap only beneath it.

## Design Considerations for Wide Icons

Custom icons wider than 20 pixels (common for logos) may render differently depending on usage context. Additional documentation addresses specific rendering behavior for wider custom icons.
